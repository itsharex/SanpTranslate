## 1. 引言

### 1.1 编写目的

本文档旨在明确 SnapTranslate 软件的功能需求、非功能需求及约束条件，为系统设计、开发、测试和验收提供统一的基准依据。本文档的预期读者包括：项目经理、系统架构师、开发工程师、测试工程师及产品相关方。

### 1.2 项目范围

SnapTranslate 是一款桌面截屏贴图翻译工具，核心能力包括：屏幕区域框选截图、截图原位贴图、剪贴板图片贴图、纯文本翻译、本地 OCR 文字提取、大模型 API 翻译、译文覆盖展示、翻译历史管理、开机自启动。软件以系统托盘常驻形态运行，所有操作通过快捷键与桌面贴图完成，不设传统主窗口。

### 1.3 术语与缩略语

| 术语/缩略语 | 定义                                                         |
|-------------|--------------------------------------------------------------|
| OCR         | 光学字符识别（Optical Character Recognition），从图像中提取文字 |
| LLM         | 大语言模型（Large Language Model）                            |
| 贴图        | 以无边框置顶窗口形式固定在桌面上的截图图像                    |
| 原位贴图    | 贴图窗口精确覆盖在用户框选的屏幕区域位置                      |
| OCR 模式    | 先本地提取文字再发送至大模型翻译的模式                        |
| tessdata    | Tesseract OCR 引擎的语言训练数据文件                          |

- 《SnapTranslate 产品需求文档（PRD）V1.7》

## 2. 总体描述

### 2.1 产品视角

SnapTranslate 是一个独立的桌面应用程序，不依赖其他软件系统。其外部交互关系如下：

- **操作系统**：通过系统 API 实现全局快捷键注册、系统托盘、剪贴板读写、窗口管理、开机自启动。
- **大模型 API**：通过 HTTPS 与用户自行配置的大模型服务通信，发送文本并接收翻译结果。
- **用户文件系统**：仅在本地存储配置文件、历史记录数据库及 OCR 语言数据，不读取其他文件。

### 2.2 用户特征

| 用户类型     | 特征描述                                                       |
|-------------|----------------------------------------------------------------|
| 学生/研究者 | 频繁阅读外语论文与资料，需要快速理解屏幕上的外语文本            |
| 职场人士    | 跨国协作中需快速翻译外语文档、截图内容，追求效率               |
| 隐私敏感用户| 拒绝截图数据被任何第三方服务获取，要求完全本地处理              |

### 2.3 运行环境

| 项目     | 最低要求                                   | 推荐配置                                   |
|----------|--------------------------------------------|--------------------------------------------|
| 操作系统 | Windows 10 (1803+) / macOS 12+ / Linux (X11/Wayland) | Windows 11 / macOS 14+ / Ubuntu 22.04+     |
| 处理器   | x64 双核 1.6GHz                            | x64 四核 2.0GHz+                           |
| 内存     | 4 GB                                       | 8 GB+                                      |
| 磁盘空间 | 300 MB（含 OCR 语言数据）                   | 500 MB                                     |
| 网络     | 需网络连接（仅与用户配置的 API 通信）       | 稳定宽带                                   |

### 2.4 约束与假设

**约束：**
- C-01：软件必须完全离线处理截图图像，仅在与用户配置的大模型 API 通信时联网。
- C-02：不得包含任何遥测、崩溃上报或用户行为追踪代码。
- C-03：API 密钥必须使用操作系统提供的安全存储机制加密保存。
- C-04：安装包须包含离线 OCR 所需资源，且体积合理（目标 < 100 MB）。

**假设：**
- A-01：用户已拥有可用的大模型 API 服务并具备基本配置能力。
- A-02：用户操作系统已安装 WebView2（Windows）或系统自带 WebKit（macOS/Linux）。
- A-03：OCR 识别精度受图像质量、字体、语言等因素影响，不保证 100% 准确。

## 3. 功能需求

### 3.1 截图功能

#### FR-SCR-001 全局截图快捷键
- **描述**：用户按下全局快捷键 `Ctrl+Alt+L` 后，进入截图模式。
- **输入**：键盘快捷键 `Ctrl+Alt+L`。
- **处理**：
  1. 注册全局快捷键监听。
  2. 检测到快捷键按下后，触发截图模式。
- **输出**：进入截图选择界面。
- **优先级**：高。
- **可定制性**：快捷键允许用户在设置中自定义。

#### FR-SCR-002 屏幕蒙版与框选
- **描述**：进入截图模式后，屏幕覆盖半透明蒙版，用户拖拽框选目标区域。
- **输入**：鼠标按下、拖拽、释放事件。
- **处理**：
  1. 捕获当前屏幕全屏图像。
  2. 在全屏图像上覆盖半透明蒙版（透明度约 50%）。
  3. 用户拖拽时实时显示选区矩形，选区内蒙版透明（显示原始画面）。
  4. 鼠标释放时确定选区坐标（x, y, width, height）。
- **输出**：确定的选区坐标及该区域的截图图像。
- **优先级**：高。

#### FR-SCR-003 截图取消
- **描述**：截图模式下按 `Esc` 键或鼠标右键取消截图。
- **输入**：键盘 `Esc` 键或鼠标右键点击。
- **处理**：关闭蒙版窗口，恢复桌面正常状态。
- **输出**：无截图产生。
- **优先级**：高。

#### FR-SCR-004 截图自动贴图
- **描述**：截图完成后，自动以无边框置顶窗口的形式贴在屏幕原位置。
- **输入**：截图图像及选区坐标。
- **处理**：
  1. 创建无边框、置顶、无任务栏图标的窗口。
  2. 窗口位置与尺寸精确匹配选区坐标。
  3. 在窗口中显示截图图像。
- **输出**：原位贴图窗口。
- **优先级**：高。

#### FR-SCR-005 截图自动复制到剪贴板
- **描述**：截图完成后，自动将截图图像复制到系统剪贴板。
- **输入**：截图图像。
- **处理**：将图像以 PNG 格式写入系统剪贴板。
- **输出**：系统剪贴板包含该截图图像。
- **优先级**：高。

### 3.2 贴图功能

#### FR-PIN-001 剪贴板贴图快捷键
- **描述**：用户按下 `Ctrl+Alt+P`，从系统剪贴板取图并贴到桌面。
- **输入**：键盘快捷键 `Ctrl+Alt+P`。
- **处理**：
  1. 读取系统剪贴板中的图像数据。
  2. 若剪贴板无图像，不响应（静默忽略）。
  3. 若有图像，生成新的置顶贴图窗口。
- **输出**：贴图窗口（默认显示在屏幕中央）。
- **优先级**：高。
- **可定制性**：快捷键允许用户在设置中自定义。

#### FR-PIN-002 贴图拖拽移动
- **描述**：用户可拖拽贴图窗口的图片区域或控制栏空白处移动窗口。
- **输入**：鼠标在贴图窗口非按钮区域的按下与拖拽事件。
- **处理**：根据鼠标偏移量实时更新窗口位置。
- **输出**：贴图窗口移动到新位置。
- **优先级**：高。

#### FR-PIN-003 贴图双击关闭
- **描述**：双击贴图图片区域关闭该贴图窗口。
- **输入**：鼠标在图片区域的双击事件。
- **处理**：销毁该贴图窗口及其资源。
- **输出**：贴图窗口关闭。
- **优先级**：高。

#### FR-PIN-004 贴图右键菜单关闭
- **描述**：右键点击贴图弹出上下文菜单，包含"关闭"选项。
- **输入**：鼠标右键点击事件。
- **处理**：显示上下文菜单，用户点击"关闭"后销毁贴图窗口。
- **输出**：贴图窗口关闭。
- **优先级**：中。

#### FR-PIN-005 多贴图并存
- **描述**：多个贴图窗口可同时存在，每个贴图独立操作。
- **输入**：多次截图或贴图操作。
- **处理**：为每次操作创建独立的贴图窗口实例。
- **输出**：多个独立的贴图窗口。
- **优先级**：高。

### 3.3 翻译功能

#### FR-TRN-001 AI 翻译触发
- **描述**：用户在贴图控制栏点击"AI翻译"按钮，启动翻译流程。
- **输入**：按钮点击事件。
- **处理**：
  1. 按钮状态变为"翻译中..."加载态。
  2. 执行 OCR 翻译流程。
- **输出**：翻译结果。
- **优先级**：高。

#### FR-TRN-002 OCR 模式翻译
- **描述**：先本地提取文字，再发送至大模型 API 翻译。
- **输入**：截图图像。
- **处理**：
  1. 将图像送入本地 Tesseract OCR 引擎。
  2. 提取文本内容及各文本块的精确位置坐标（bounding box）。
  3. 查找历史缓存（根据 OCR 文本和目标语言匹配）。
  4. 若命中缓存，直接返回翻译结果（跳过 API 调用）。
  5. 若未命中缓存，将识别出的文本按顺序组织，构造翻译请求。
  6. 发送至用户配置的大模型 API。
  7. 解析返回结果，将译文映射回各文本块。
- **输出**：各文本块的原文、译文及坐标信息。
- **优先级**：高。

#### FR-TRN-002-1 翻译缓存机制
- **描述**：自动匹配历史翻译记录，相同内容跳过 API 调用。
- **输入**：OCR 识别文本、目标语言。
- **处理**：
  1. 根据 OCR 文本和目标语言查询历史记录数据库。
  2. 若找到匹配记录，直接返回缓存的翻译结果。
  3. 若未找到，执行正常的 API 翻译流程。
  4. 翻译完成后，将结果（含 target_language 和 blocks_json）保存到历史记录。
- **输出**：翻译结果（含 from_cache 标志）。
- **优先级**：中。

#### FR-TRN-003 翻译失败处理
- **描述**：翻译过程中出现错误时给出明确提示。
- **输入**：API 错误响应、网络超时、OCR 失败等。
- **处理**：
  1. 捕获异常并记录日志。
  2. 按钮恢复为"AI翻译"状态。
  3. 显示错误提示信息（如"网络连接失败"、"API 返回错误"等）。
- **输出**：错误提示。
- **优先级**：中。

### 3.4 文本翻译功能

#### FR-TXT-001 文本翻译快捷键
- **描述**：用户按下 `Ctrl+Alt+M`，打开文本翻译窗口。
- **输入**：键盘快捷键 `Ctrl+Alt+M`。
- **处理**：
  1. 检测到快捷键按下后，创建文本翻译窗口。
  2. 若窗口已存在，聚焦现有窗口。
- **输出**：文本翻译窗口显示在屏幕下方居中位置。
- **优先级**：高。
- **可定制性**：快捷键允许用户在设置中自定义。

#### FR-TXT-002 文本翻译输入
- **描述**：用户在文本翻译窗口输入或粘贴要翻译的文本。
- **输入**：用户输入的文本。
- **处理**：
  1. 显示多行文本输入框。
  2. 支持 `Ctrl+Enter` 快捷键触发翻译。
- **输出**：输入的文本内容。
- **优先级**：高。

#### FR-TXT-003 文本翻译执行
- **描述**：用户点击翻译按钮或按 `Ctrl+Enter`，执行文本翻译。
- **输入**：输入的文本、目标语言。
- **处理**：
  1. 查找历史缓存（根据输入文本和目标语言匹配）。
  2. 若命中缓存，直接返回翻译结果（跳过 API 调用）。
  3. 若未命中缓存，调用大模型 API 翻译。
  4. 异步保存翻译历史（无图片）。
- **输出**：翻译后的文本（含 from_cache 标志）。
- **优先级**：高。

#### FR-TXT-004 文本翻译窗口关闭
- **描述**：用户可通过点击关闭按钮或按 `Esc` 键关闭文本翻译窗口。
- **输入**：关闭按钮点击或 `Esc` 键按下。
- **处理**：销毁文本翻译窗口。
- **输出**：窗口关闭。
- **优先级**：中。

### 3.5 翻译结果展示

#### FR-DSP-001 译文面板显示
- **描述**：翻译完成后，译文显示在贴图右侧的独立面板中。
- **输入**：各文本块的译文。
- **处理**：
  1. 在贴图右侧创建译文面板。
  2. 面板中按顺序显示各翻译块。
  3. 面板支持高度拉伸。
- **输出**：右侧译文面板。
- **优先级**：高。

#### FR-DSP-002 复制原文
- **描述**：点击控制栏"复制原文"按钮，复制所有原文到剪贴板。
- **输入**：按钮点击事件。
- **处理**：将所有文本块的原文合并为一段文本，写入系统剪贴板。
- **输出**：剪贴板包含全部原文。
- **优先级**：中。

#### FR-DSP-003 复制译文
- **描述**：点击控制栏"复制译文"按钮，复制所有译文到剪贴板。
- **输入**：按钮点击事件。
- **处理**：将所有文本块的译文合并为一段文本，写入系统剪贴板。
- **输出**：剪贴板包含全部译文。
- **优先级**：中。

#### FR-DSP-004 原文/译文切换
- **描述**：点击控制栏"原文/译文"切换按钮，在显示原文与显示译文面板之间切换。
- **输入**：按钮点击事件。
- **处理**：
  1. 切换时隐藏译文面板，显示图片原文。
  2. 再次切换恢复译文面板显示。
- **输出**：原文或译文的显示状态。
- **优先级**：高。

#### FR-DSP-005 重新翻译
- **描述**：点击控制栏"重新翻译"按钮，强制跳过缓存重新调用 API 翻译。
- **输入**：按钮点击事件。
- **处理**：
  1. 设置 `force_retranslate` 为 true。
  2. 重新执行 OCR 和翻译流程。
- **输出**：新的翻译结果。
- **优先级**：中。

### 3.6 API 配置管理

#### FR-CFG-001 设置入口
- **描述**：通过系统托盘右键菜单的"设置"项打开设置界面。
- **输入**：菜单项点击事件。
- **处理**：打开设置窗口。
- **输出**：设置界面。
- **优先级**：高。

#### FR-CFG-002 API 配置项
- **描述**：设置界面需包含以下配置项：
  - 界面语言（下拉列表选择：跟随系统/简体中文/English）
  - 通用设置（开机自启动开关）
  - API 基础地址（必填）
  - API 密钥（必填，安全输入，可切换显示/隐藏，可删除）
  - 模型名称（必填）
  - 目标翻译语言（下拉列表选择）
- **输入**：用户填写的配置数据。
- **处理**：验证配置数据格式，保存至本地加密配置文件。
- **输出**：配置保存成功/失败提示。
- **优先级**：高。

#### FR-CFG-003 安全存储
- **描述**：API 密钥使用操作系统提供的安全存储机制加密保存。
- **输入**：API 密钥明文。
- **处理**：调用操作系统凭据管理器（Windows Credential Manager / macOS Keychain / Linux Secret Service）加密存储。
- **输出**：加密后的密钥存储。
- **优先级**：高。

#### FR-CFG-004 连接测试
- **描述**：提供"测试连接"按钮，发送最小请求验证 API 可用性。
- **输入**：按钮点击事件。
- **处理**：
  1. 使用当前配置构造最小 API 请求（如发送简单文本"Hello"请求翻译）。
  2. 发送请求并等待响应。
  3. 根据响应状态反馈成功或失败。
- **输出**：连接测试结果提示。
- **优先级**：中。

#### FR-CFG-005 快捷键自定义
- **描述**：用户可在设置中自定义所有全局快捷键。
- **输入**：用户输入的新快捷键组合。
- **处理**：
  1. 注销旧快捷键注册。
  2. 注册新快捷键。
  3. 保存到配置文件。
- **输出**：快捷键更新成功/失败提示。
- **优先级**：中。

#### FR-CFG-006 删除 API 密钥
- **描述**：用户可在设置中删除已保存的 API 密钥。
- **输入**：删除按钮点击事件。
- **处理**：
  1. 显示确认对话框。
  2. 用户确认后，从操作系统凭据管理器中删除密钥。
- **输出**：密钥删除成功/失败提示。
- **优先级**：中。

#### FR-CFG-007 配置文件路径显示
- **描述**：设置界面显示配置文件的完整路径。
- **输入**：页面加载事件。
- **处理**：读取并显示配置文件路径。
- **输出**：配置文件路径文本。
- **优先级**：低。

### 3.7 系统托盘

#### FR-TRY-001 托盘常驻
- **描述**：软件启动后在系统托盘显示图标并常驻。
- **输入**：软件启动事件。
- **处理**：创建系统托盘图标及右键菜单。
- **输出**：系统托盘图标可见。
- **优先级**：高。

#### FR-TRY-002 托盘菜单
- **描述**：托盘右键菜单包含以下项：
  - 框选截图翻译（`Ctrl+Alt+L`）
  - 从剪贴板贴图（`Ctrl+Alt+P`）
  - 文本翻译（`Ctrl+Alt+M`）
  - 截图与翻译历史
  - 设置
  - 重新启动
  - 退出
- **输入**：右键点击托盘图标。
- **处理**：显示菜单，处理菜单项点击事件。
- **输出**：对应功能触发。
- **优先级**：高。

#### FR-TRY-003 托盘菜单语言切换
- **描述**：界面语言切换后，托盘菜单文本自动更新为对应语言。
- **输入**：语言配置变更事件。
- **处理**：重新构建托盘菜单，使用新语言的文本。
- **输出**：更新后的托盘菜单。
- **优先级**：中。

### 3.8 界面语言

#### FR-LANG-001 自动跟随系统语言
- **描述**：软件界面文字自动跟随操作系统语言切换，最低支持中文与英文。
- **输入**：操作系统语言设置。
- **处理**：启动时检测系统语言，加载对应语言包。
- **输出**：对应语言的界面文字。
- **优先级**：中。

#### FR-LANG-002 语言即时切换
- **描述**：用户在设置中切换界面语言后，立即生效，无需点击保存按钮。
- **输入**：语言下拉列表选择事件。
- **处理**：
  1. 立即更新当前窗口的语言。
  2. 保存语言配置到后端。
  3. 后端广播语言变更事件到所有窗口。
  4. 更新托盘菜单文本。
- **输出**：所有窗口和托盘菜单使用新语言。
- **优先级**：中。

### 3.9 开机自启动

#### FR-AUTO-001 开机自启动设置
- **描述**：用户可在设置中开启或关闭开机自启动功能。
- **输入**：开关切换事件。
- **处理**：
  1. 调用操作系统 API 注册或取消注册开机启动项。
  2. 保存设置状态。
- **输出**：开机自启动状态变更提示。
- **优先级**：低。

### 3.10 历史记录

#### FR-HIS-001 自动记录翻译历史
- **描述**：通过 `Ctrl+Alt+L` 截图并完成翻译后，自动保存历史项。
- **输入**：翻译完成事件。
- **处理**：
  1. 生成截图缩略图。
  2. 保存原图数据（Base64 编码）。
  3. 保存 OCR 识别文本、翻译后文本、目标语言、翻译块 JSON、时间戳。
  4. 写入本地 SQLite 数据库。
- **输出**：历史记录条目。
- **优先级**：中。

#### FR-HIS-002 文本翻译历史记录
- **描述**：文本翻译完成后，自动保存历史项（无图片）。
- **输入**：文本翻译完成事件。
- **处理**：
  1. 保存输入文本、翻译后文本、目标语言、翻译块 JSON、时间戳。
  2. 图片相关字段为空。
  3. 写入本地 SQLite 数据库。
- **输出**：历史记录条目（无缩略图）。
- **优先级**：中。

#### FR-HIS-003 历史面板查看
- **描述**：通过托盘菜单"截图与翻译历史"打开历史面板。
- **输入**：菜单项点击事件。
- **处理**：从数据库读取最近 50 条历史记录并展示。
- **输出**：历史记录列表（含缩略图、翻译摘要、时间）。
- **优先级**：中。

#### FR-HIS-004 历史详情查看与复制
- **描述**：点击某条历史记录可查看详细信息并复制翻译文本。
- **输入**：历史条目点击事件。
- **处理**：展示完整原文与译文，提供复制按钮。
- **输出**：详细信息视图。
- **优先级**：低。

#### FR-HIS-005 历史删除
- **描述**：支持逐条删除或一键清空全部历史。
- **输入**：删除/清空操作。
- **处理**：从数据库中删除对应记录。
- **输出**：删除确认与结果。
- **优先级**：中。

#### FR-HIS-006 贴图不产生历史
- **描述**：通过 `Ctrl+Alt+P` 单纯贴出的图片若未进行翻译，不自动进入历史。
- **输入**：贴图操作。
- **处理**：仅在翻译完成后触发历史记录。
- **输出**：无历史条目产生。
- **优先级**：中。

| 目标翻译语言     | Enum     | 本地配置文件      | 否    | 中文/英文/日文等          |
